## 1. 核心叙事

### 1.1 故事背景

一颗来自濒死恒星的「种子」——宇宙中最后一份生命火种——被托付给一位星际守护者（玩家）。守护者驾驶列车穿越星海，护送种子寻找传说中的「生命之源」——一颗能让种子生根发芽、重获新生的宜居星球。

### 1.2 剧情结构

```
三幕式结构：
  第一幕「启程」— 种子刚苏醒，懵懂不安，学习与守护者沟通
  第二幕「成长」— 经历星球抉择，种子性格分化，成长方向由选择决定
  第三幕「绽放」— 接近终点，种子面临最终考验，准备扎根新世界
```

### 1.3 主题映射

| 科普知识点 | 游戏化表达 |
|-----------|-----------|
| 种子传播机制 | 星际航行中抵抗陨石、适应环境 |
| 光合作用 / 养分吸收 | 生命活性产出，吸收星光 |
| 环境适应性 | 星球抉择影响成长方向 |
| 生命周期 | 种子 → 发芽 → 生长 → 开花 → 结果 |
| 生态系统平衡 | 保护种子与列车之间的关系 |

### 1.4 种子对话主题库（扩展")

| 阶段 | 对话主题 | 科普内容 |
|------|---------|---------|
| 启程 | 我是谁？我从哪里来？ | 恒星生命周期、元素合成 |
| 成长 | 宇宙中还有别的生命吗？ | 系外行星、宜居带 |
| 成长 | 为什么要有根？ | 植物根系、养分吸收 |
| 成长 | 你听过光合作用吗？ | 光能→化学能转化 |
| 绽放 | 什么是家？ | 生态系统、栖息地 |

## 2. 星球抉择系统

### 2.1 触发方式

- 每航行一段**固定距离**（或击杀一定数量敌人后）触发
- 列车前方出现大型星球，摄像头拉远，展示 2~3 个星球选项
- 游戏暂停，展示星球卡片（名称 + 属性加成 + 视觉预览）

### 2.2 星球类型

| 星球 | 种子成长方向 | 永久 Buff | 视觉特征 |
|------|-------------|-----------|---------|
| 水之星 · 澜 | 柔韧滋养 | 生命活性产出 +20% | 蓝色荧光，波浪纹理 |
| 火之星 · 烬 | 力量爆发 | 武器伤害 +15% | 红色熔岩，脉冲光效 |
| 森之星 · 栖 | 坚韧守护 | 列车 HP +30，安全区 +15% | 绿色藤蔓，叶片环绕 |
| 光之星 · 辉 | 均衡协调 | 所有属性 +8% | 金色光芒，光环扩散 |
| 暗之星 · 寂 | 隐秘专注 | 种子对话冷却 -40%，buff 持续时间 ×1.5 | 紫色星云，粒子拖尾 |

### 2.3 选择流程

1. 进入星球选择场景（自动，游戏暂停）
2. 显示 2~3 个星球，每个带预览图和属性说明
3. 玩家鼠标点击选择一个 → 确认动画 → 应用 buff → 继续航行
4. 种子外观根据累积选择发生变化（见第 4 节）

### 2.4 选择影响

- **短期**：立即获得该星球的烙印 buff
- **长期**：种子外观积累发生变化（水→蓝、火→红、森→绿、光→金、暗→紫）
- **种子性格**：星球选择倾向影响对话系统的概率分布

## 3. 背包与物品系统

### 3.1 物品获取

| 来源 | 获取方式 |
|------|---------|
| 特殊陨石（金色闪烁） | 安魂曲或导弹击碎后掉落 |
| 星球抉择 | 选择某星球后赠送专属物品 |
| 种子对话奖励 | 正确选项偶尔掉落特殊物品 |
| 事件奖励 | 航行事件选择正确时获得 |

### 3.2 物品类型

| 类别 | 名称 | 效果 | 稀有度 |
|------|------|------|-------|
| 消耗品 | 星尘 | 使用后 +50 货币 | 普通 |
| 消耗品 | 能量水晶 | 使用后 10s 内移速 ×1.5 + 免伤 | 稀有 |
| 消耗品 | 生命甘露 | 种子 HP +20 | 稀有 |
| 材料 | 星核碎片 | 合成高级物品用，不可直接使用 | 稀有 |
| 材料 | 灵魂花瓣 | 种子进化用 | 史诗 |
| 特殊 | 古老种子 | 解锁专属 AI 对话内容 | 传说 |

### 3.3 背包 UI

- 按 `B` 键打开/关闭背包（游戏暂停）
- 网格布局（4×4 格），显示物品图标 + 数量
- 鼠标悬停显示物品名称和描述
- 左键点击消耗品 → 使用确认 → 效果生效
- 背包容量有限（默认 12 格），超限时新物品落地成拾取物

### 3.4 物品系统接口

```typescript
interface InventoryItem {
  id: string;
  name: string;
  description: string;
  rarity: 'common' | 'rare' | 'epic' | 'legendary';
  icon: string; // CSS 字符或 emoji
  stackable: boolean;
  maxStack: number;
  onUse?: (game: Game) => void;
}

class InventorySystem {
  items: Map<string, { item: InventoryItem; count: number }>;
  maxSlots: number;
  addItem(item: InventoryItem, count: number): boolean;
  removeItem(id: string, count: number): boolean;
  useItem(id: string): boolean;
  hasItem(id: string): boolean;
}
```

## 4. 种子成长视觉

### 4.1 成长阶段

| 阶段 | 累积星球抉择次数 | 视觉表现 |
|------|----------------|---------|
| 萌芽期 | 0 | 小圆点 + 微弱光晕 |
| 成长期 | 1 | 两片叶子（Geometry 组合），缓慢旋转 |
| 繁茂期 | 2 | 多层叶片 + 花苞，粒子环绕 |
| 开花期 | 3 | 花朵绽放 + 花粉粒子飘散 |
| 结果期 | 4 | 果实 + 强光晕，全屏轻微花瓣粒子 |

### 4.2 基于星球抉择的视觉分化

累积选择偏好触发外观变化：

| 偏好 | 颜色 | 特效 |
|------|------|------|
| 水之星 | #4fc3f7 | 波浪粒子环绕 |
| 火之星 | #ff5252 | 火花粒子升腾 |
| 森之星 | #69f0ae | 叶片飘落 |
| 光之星 | #ffd740 | 金光粒子四射 |
| 暗之星 | #b39ddb | 星云粒子拖尾 |

所有阶段和颜色通过 Three.js `Mesh` + `Sprite` + `PointsMaterial` 组合实现。

### 4.3 种子车厢视觉更新

种子车厢内显示种子本体，成长阶段越高 → 种子车厢外观越华丽（发光、藤蔓装饰、光晕范围扩大）。

## 5. 音效系统

### 5.1 技术方案

- 使用 **Web Audio API**（`AudioContext` + `OscillatorNode` + `GainNode`）
- 所有音效程序化生成，无需外部音频文件
- 全局 `AudioManager` 管理音量和播放
- 在 `Game.ts` 或独立 `AudioManager.ts` 中实现

### 5.2 音效列表

| 触发事件 | 音效设计 | 频率/时长 |
|---------|---------|----------|
| 激光开火 | 持续高频振荡 + 白噪声 | 800Hz → 1200Hz 扫描，持续射击 |
| 激光命中 | 短促爆裂声 | 200Hz 噪声脉冲，0.1s |
| 吸盘捕获 | 低频嗡鸣上升 | 100Hz → 300Hz，0.5s |
| 吸盘发射 | 低频冲击声 | 80Hz 脉冲 + 噪声衰减，0.3s |
| 安魂曲射击 | 深沉爆音 + 混响 | 60Hz 脉冲 + 噪声尾音，0.4s |
| 安魂曲命中 | 清脆碎裂 | 400Hz 谐波 + 噪声爆炸，0.2s |
| 导弹发射 | 呼啸上升 | 200Hz → 800Hz 扫描，0.8s |
| 导弹爆炸 | 低频轰鸣 | 40Hz 轰隆 + 噪声衰减，1.0s |
| 种子对话 | 柔和风铃 | 600Hz / 800Hz / 1000Hz 琶音，0.6s |
| 获取货币 | 短促叮当 | 1200Hz 短脉冲，0.05s |
| UI 交互 | 点击反馈 | 1000Hz 短促响应，0.03s |
| 背景氛围 | 低频空间嗡鸣 | 55Hz 正弦波 + 缓慢 LFO，循环 |
| 星球选择 | 宏大和弦 | C 大三和弦琶音，1.5s |

### 5.3 音量控制

- HUD 右上角新增音量滑块（或简单 +/- 按钮）
- 全局增益节点控制主音量（0~1）

### 5.4 AudioManager 接口

```typescript
class AudioManager {
  private ctx: AudioContext;
  private masterGain: GainNode;
  
  play(id: SoundId): void;
  setVolume(v: number): void;
  resume(): void; // 处理用户交互后恢复 AudioContext
}

type SoundId = 
  | 'laser_fire' | 'laser_hit' 
  | 'suction_catch' | 'suction_launch'
  | 'requiem_fire' | 'requiem_hit'
  | 'missile_launch' | 'missile_explode'
  | 'dialogue_chime'
  | 'coin_pickup' | 'ui_click'
  | 'ambient';

```

## 6. AI 对话系统

### 6.1 方案选择

**推荐方案：离线关键词响应 + 可选在线 AI**

第一阶段先实现本地关键词匹配系统，后续可接入在线 AI API（如 DeepSeek 或通义千问）。

### 6.2 本地 AI 模式

- 种子已有的 4 种性格（开朗/忧郁/傲娇/沉默）保持不变
- 当固定对话耗尽后，启用**动态关键词响应系统**
- 玩家在对话面板中输入关键词或选择预设选项
- 系统根据关键词 + 种子当前性格 + 成长阶段 → 映射到模板响应池

响应模板格式：
```
{
  keywords: ['生命', '活着', '存在'],
  responses: [
    { text: '生命就像星光，虽然渺小，但我们都在努力发光。', buff: BuffType.SPEED },
    { text: '你知道一粒沙里有多少个原子吗？就像宇宙里的星星一样多。', buff: BuffType.DAMAGE },
  ]
}
```

### 6.3 在线 AI 模式（加分项）

- 接入 DeepSeek API（或其他免费/低成本 LLM API）
- 系统 prompt 定义种子性格 + 当前成长阶段 + 科普主题约束
- 用户自由输入文字，AI 实时回复
- API Key 通过 `.env` 配置，不硬编码

### 6.4 AI 对话 UI

- 对话面板扩展为聊天界面样式
- 左侧展示种子头像（根据成长阶段变化）
- 右侧消息气泡（种子回复显示在左，玩家选项/输入在右）
- 在线模式下增加文本输入框

## 7. 技术方案

### 7.1 实现优先级

| 优先级 | 系统 | 预估工时 | 前置依赖 |
|--------|------|---------|---------|
| P0 | 种子成长视觉 | 2h | 无 |
| P0 | 星球抉择系统 | 4h | 无 |
| P1 | 音效系统 | 3h | 无 |
| P1 | 背包/物品系统 | 4h | 安魂曲（已有） |
| P2 | AI 对话（本地） | 3h | 对话系统（已有） |
| P3 | AI 对话（在线） | 4h | 本地 AI + 网络请求封装 |

### 7.2 新增文件

```
src/
├── systems/
│   ├── PlanetSelectionSystem.ts   # 星球抉择
│   ├── InventorySystem.ts         # 背包系统
│   ├── AudioManager.ts            # 音效管理
│   └── AIDialogueSystem.ts        # AI 对话系统（本地 + 在线）
├── entities/
│   └── SeedVisuals.ts             # 种子成长视觉系统
├── ui/
│   ├── InventoryPanel.ts          # 背包 UI
│   └── PlanetSelectionPanel.ts    # 星球选择 UI
└── config/
    └── planets.ts                 # 星球数据配置
```

### 7.3 数据流

```
Game.update():
  1. 航行距离追踪 → 触发星球抉择
  2. 特殊陨石粉碎 → InventorySystem.addItem
  3. 种子对话完成 → AIDialogueSystem.respond
  4. 种子成长检查 → SeedVisuals.update(阶段)
  5. 武器/交互事件 → AudioManager.play
```
