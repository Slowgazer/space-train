# 网页游戏开发项目上下文

> 创建时间: 2026-06-09
> 开发者备忘录 — 每次开发前请先阅读此文件

---

## 一、核心需求（不可遗忘）

1. **网页游戏** — 基于 HTML5 Canvas / WebGL，其他设备浏览器可直接访问（非原生应用）
2. **本地优先** — 第一阶段仅开发单机可运行的游戏本体
3. **预留联机接口** — 代码架构必须为后续多人联机功能留好扩展点
4. **跨设备访问** — 开发模式下局域网内其他设备可访问（需配置 host）
5. **后续迭代友好** — 网络层需抽象，后续可无缝切换本地 ↔ 远程

## 三、API / 网络层架构要求

所有游戏网络操作必须通过抽象接口，不能直接硬编码本地逻辑。

```
src/network/
├── types.ts          # 网络接口类型定义（所有数据包格式）
├── GameNetwork.ts    # 抽象接口（待后续实现远程版）
├── LocalNetwork.ts   # 本地实现（单机调试用）
├── RemoteNetwork.ts  # 远程实现（多人联机时实现）
└── index.ts          # 工厂函数，根据环境返回对应实例
```

**游戏逻辑只依赖 `GameNetwork` 接口**，不直接引用 `LocalNetwork` 或 `RemoteNetwork`。

## 四、开发路线图（初步）

1. **讨论确定游戏内容** → 输出游戏设计文档（`docs/game-design.md`）
2. **搭建项目骨架** → Vite + TypeScript + 网络抽象层
3. **实现游戏引擎** → 游戏循环、渲染、输入、物理
4. **实现游戏内容** → 根据设计文档开发具体游戏
5. **测试多人联机**（后续） → 实现 RemoteNetwork + 后端

## 五、环境配置说明

| 用途 | 文件 | 说明 |
|------|------|------|
| API 基础地址 | `.env` | `VITE_API_BASE_URL=http://localhost:3000/api` |
